## What Was Actually Wrong (and what I fixed)

Your upload contained **two mobile app folders**:
- `debale-mobile/` — the complete, correct, up-to-date one (kept)
- `debale-mobile 1/` — an older, abandoned draft with duplicate `.js`/`.jsx` files for
  the same screens and **no** housemate feature at all (removed)

This is almost certainly why it looked like the mobile app was missing the feature —
if you'd opened `debale-mobile 1` instead of `debale-mobile`, it would look unbuilt.

I also stripped `node_modules`, `.expo`, and `.expo-test` folders from everywhere in the
zip — those get regenerated by `npm install` and were bloating the file from ~1.2GB down
to under 4MB with zero loss of code.

## Folder Structure Now

```
debale-backend/     ← Node.js + Express + Supabase API (includes housemate.js + migration_housemate.sql)
debale-web/          ← React web app (includes all Housemate*.jsx pages)
debale-mobile/        ← React Native + Expo app (includes all Housemate*Screen.js + Group tab)
```

## How to Run Each Part

**Backend:**
```
cd debale-backend
npm install
npm run dev
```
Make sure you've run both `schema.sql` and `migration_housemate.sql` in your Supabase SQL Editor.

**Web:**
```
cd debale-web
npm install --legacy-peer-deps
npx vite
```

**Mobile:**
```
cd debale-mobile
npm install --legacy-peer-deps
npx expo start
```
Set your backend LAN IP in `src/services/api.js` before running on a physical phone.
